# examples.md : axum-impl-file-upload

Working, version-annotated examples for Axum file uploads. Every API call was
verified against the official `docs.rs` pages on 2026-05-20. The `Multipart`
and `Field` APIs are identical in Axum 0.7 and 0.8, so these examples compile
on both lines; only the route path syntax differs between versions, and the
upload routes here use static paths, which are unaffected.

## Example 1: complete buffering upload server

Suitable for small fields only. Every field is read fully into memory with
`bytes()`. The verbatim official handler shape, wrapped in a runnable server.

```rust
// Cargo.toml
// axum  = { version = "0.8", features = ["multipart"] }
// tokio = { version = "1",   features = ["full"] }

use axum::{extract::Multipart, routing::post, Router};

async fn upload(mut multipart: Multipart) {
    while let Some(mut field) = multipart.next_field().await.unwrap() {
        let name = field.name().unwrap().to_string();
        let data = field.bytes().await.unwrap();

        println!("Length of `{}` is {} bytes", name, data.len());
    }
}

#[tokio::main]
async fn main() {
    let app = Router::new().route("/upload", post(upload));

    let listener = tokio::net::TcpListener::bind("0.0.0.0:3000")
        .await
        .unwrap();
    axum::serve(listener, app).await.unwrap();
}
```

`axum::serve` is the 0.7 and 0.8 server entry point. The `.unwrap()` calls are
acceptable only for a demo; Example 4 shows production error handling.

## Example 2: mixed text-and-file form

A real form often carries text fields and a file together. Read each field,
branch on `file_name()` to tell a file part from a plain text part.

```rust
use axum::{extract::Multipart, http::StatusCode};

async fn submit(mut multipart: Multipart) -> Result<String, (StatusCode, String)> {
    let mut caption = String::new();
    let mut file_bytes: Option<Vec<u8>> = None;

    while let Some(field) = multipart
        .next_field()
        .await
        .map_err(|e| (StatusCode::BAD_REQUEST, e.to_string()))?
    {
        // Read metadata BEFORE consuming the field body.
        let field_name = field.name().unwrap_or_default().to_string();
        let is_file = field.file_name().is_some();

        if is_file {
            let bytes = field
                .bytes()
                .await
                .map_err(|e| (StatusCode::BAD_REQUEST, e.to_string()))?;
            file_bytes = Some(bytes.to_vec());
        } else if field_name == "caption" {
            caption = field
                .text()
                .await
                .map_err(|e| (StatusCode::BAD_REQUEST, e.to_string()))?;
        }
    }

    match file_bytes {
        Some(b) => Ok(format!("caption={caption}, file={} bytes", b.len())),
        None => Err((StatusCode::BAD_REQUEST, "no file field".to_string())),
    }
}
```

## Example 3: stream a large file to disk with a safe destination

Files are streamed chunk by chunk so peak memory stays at one chunk. The
on-disk name is derived by the server, never taken from `file_name()`.

```rust
use axum::extract::{multipart::Field, Multipart};
use axum::http::StatusCode;
use std::path::{Path, PathBuf};
use std::time::{SystemTime, UNIX_EPOCH};
use tokio::fs::File;
use tokio::io::AsyncWriteExt;

const UPLOAD_DIR: &str = "/var/uploads";

/// Build a safe destination path. The client file name is used ONLY to
/// recover an extension; the stem is generated by the server.
fn safe_dest(client_name: Option<&str>) -> PathBuf {
    let ext = client_name
        .and_then(|n| Path::new(n).extension())
        .and_then(|e| e.to_str())
        .filter(|e| e.chars().all(|c| c.is_ascii_alphanumeric()))
        .unwrap_or("bin");

    let stamp = SystemTime::now()
        .duration_since(UNIX_EPOCH)
        .unwrap()
        .as_nanos();

    Path::new(UPLOAD_DIR).join(format!("upload-{stamp}.{ext}"))
}

async fn stream_field_to_disk(mut field: Field<'_>) -> std::io::Result<PathBuf> {
    let dest = safe_dest(field.file_name());
    let mut file = File::create(&dest).await?;

    while let Some(chunk) = field
        .chunk()
        .await
        .map_err(|e| std::io::Error::new(std::io::ErrorKind::InvalidData, e))?
    {
        file.write_all(&chunk).await?;
    }
    file.flush().await?;
    Ok(dest)
}

async fn upload(mut multipart: Multipart) -> Result<String, (StatusCode, String)> {
    let mut saved = Vec::new();
    while let Some(field) = multipart
        .next_field()
        .await
        .map_err(|e| (StatusCode::BAD_REQUEST, e.to_string()))?
    {
        if field.file_name().is_some() {
            let path = stream_field_to_disk(field)
                .await
                .map_err(|e| (StatusCode::INTERNAL_SERVER_ERROR, e.to_string()))?;
            saved.push(path.display().to_string());
        }
    }
    Ok(format!("saved: {saved:?}"))
}
```

A UUID stem (via the `uuid` crate) is the strongest choice when collisions
must be impossible; the timestamp stem above keeps the example dependency
free. Either way, the server controls the name.

## Example 4: raise the body limit per endpoint

The 2 MB default rejects real uploads. Raise it with `DefaultBodyLimit::max`.

```rust
use axum::{extract::DefaultBodyLimit, routing::post, Router};

fn upload_router() -> Router {
    Router::new()
        .route("/upload", post(upload))
        // 25 MB ceiling for this endpoint; replaces the 2 MB default.
        .layer(DefaultBodyLimit::max(25 * 1024 * 1024))
}
# async fn upload() {}
```

The official `DefaultBodyLimit` example, for reference:

```rust
use axum::{body::Bytes, extract::DefaultBodyLimit, routing::get, Router};

let app: Router<()> = Router::new()
    .route("/", get(|body: Bytes| async {}))
    // Replace the default of 2MB with 1024 bytes.
    .layer(DefaultBodyLimit::max(1024));
```

## Example 5: disable the default and set a tower-http hard ceiling

Use this only when the `tower-http` limiter is specifically required. The two
layers MUST appear together.

```rust
// Cargo.toml
// tower-http = { version = "0.6", features = ["limit"] }

use axum::{extract::DefaultBodyLimit, routing::post, Router};
use tower_http::limit::RequestBodyLimitLayer;

fn upload_router() -> Router {
    Router::new()
        .route("/upload", post(upload))
        .layer(DefaultBodyLimit::disable())
        // Hard ceiling: oversized bodies become 413 Payload Too Large.
        .layer(RequestBodyLimitLayer::new(100 * 1024 * 1024)) // 100 MB
}
# async fn upload() {}
```

Removing `RequestBodyLimitLayer` from this example makes the endpoint accept
an unbounded body, which is a memory-exhaustion and disk-exhaustion DoS
vector. NEVER ship `disable()` alone.

## Example 6: validate content by inspecting bytes, not the declared type

`content_type()` is client-supplied. To accept only PNG files, check the
magic bytes of the buffered data instead of trusting the header.

```rust
use axum::{extract::Multipart, http::StatusCode};

const PNG_MAGIC: &[u8] = &[0x89, 0x50, 0x4E, 0x47, 0x0D, 0x0A, 0x1A, 0x0A];

async fn upload_png(mut multipart: Multipart) -> Result<String, (StatusCode, String)> {
    while let Some(field) = multipart
        .next_field()
        .await
        .map_err(|e| (StatusCode::BAD_REQUEST, e.to_string()))?
    {
        if field.file_name().is_some() {
            let data = field
                .bytes()
                .await
                .map_err(|e| (StatusCode::BAD_REQUEST, e.to_string()))?;

            if !data.starts_with(PNG_MAGIC) {
                return Err((StatusCode::BAD_REQUEST, "not a PNG file".to_string()));
            }
            return Ok(format!("accepted PNG, {} bytes", data.len()));
        }
    }
    Err((StatusCode::BAD_REQUEST, "no file field".to_string()))
}
```

For a streamed upload, inspect the first chunk before writing the rest, or
validate after the file is on disk and delete it if validation fails.
